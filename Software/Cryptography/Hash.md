# üîê Hashing in Cryptography

## üß© What is a Hash?

A **hash** is a **fixed-length string of characters** (a digest) generated by applying a **hash function** to data of any size.

Hashing is a **one-way transformation** ‚Äî once data is hashed, it cannot be reversed to obtain the original message.

---

## ‚öôÔ∏è Purpose of Hashing

| Use Case | Description |
|-----------|-------------|
| **Data integrity** | Verify data hasn‚Äôt been changed (e.g., file checksums). |
| **Password storage** | Store hashes instead of plain-text passwords. |
| **Digital signatures** | Hash used to ensure authenticity and integrity. |
| **Blockchain** | Links blocks together using previous block hashes. |

---

## üß† How a Hash Function Works

A hash function takes input data and outputs a fixed-size value.

Example:

```
Input: "hello"  
SHA-256: 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
```


‚úÖ Key properties:
- Same input ‚Üí same hash (deterministic)
- Small change in input ‚Üí drastically different hash (avalanche effect)
- Impossible to get original input from hash (one-way)
- Very low probability of collisions (two inputs with same hash)

---

## üîë Important Hash Properties

| Property                       | Meaning                                                   |
| ------------------------------ | --------------------------------------------------------- |
| **Deterministic**              | Same input always gives the same output.                  |
| **Fast computation**           | Efficiently computable.                                   |
| **Preimage resistance**        | Cannot derive input from its hash.                        |
| **Second preimage resistance** | Hard to find another input with the same hash.            |
| **Collision resistance**       | Hard to find two different inputs with identical hashes.  |
| **Avalanche effect**           | Tiny input change ‚Üí large unpredictable change in output. |

---

## üßÆ Common Cryptographic Hash Functions

| Algorithm | Output Size | Notes |
|------------|--------------|-------|
| **MD5** | 128 bits | Fast but **not secure** (collision attacks). |
| **SHA-1** | 160 bits | Deprecated (broken). |
| **SHA-256** | 256 bits | Secure, widely used in SSL, Bitcoin, etc. |
| **SHA-512** | 512 bits | Stronger but larger output. |
| **SHA-3 (Keccak)** | Variable | Newest SHA standard. |
| **BLAKE2 / BLAKE3** | Variable | Modern, faster alternatives to SHA. |

## üîê Hashing vs Encryption

| Feature | Hashing | Encryption |
|----------|----------|-------------|
| Direction | One-way | Two-way (reversible with key) |
| Purpose | Integrity / fingerprinting | Confidentiality |
| Output | Fixed size | Variable size (same as input) |
| Key used | No | Yes |
| Example | SHA-256 | AES, RSA |

## üß± Hashing in Password Storage

Instead of storing passwords directly:

1. Generate a **salt** (random string).
2. Combine it with the password ‚Üí `password + salt`.
3. Hash the combination.
4. Store the hash and salt.

This prevents rainbow table attacks.

Example:
```
Password: mySecret!  
Salt: ab12cd  
Stored hash: SHA-256("mySecret!ab12cd")
```


### üîí Secure Hashing Algorithms for Passwords
Use specialized functions that include salting and key stretching:
- **bcrypt**
- **scrypt**
- **PBKDF2**
- **Argon2** (recommended modern choice)

---

## üß† Hash Collisions

A **collision** occurs when two different inputs produce the same hash.  
A good hash function makes this extremely difficult.

Example (theoretically):
```
H("Hello") = abc123...  
H("World!") = abc123... ‚Üê collision
```


Cryptographically secure hash functions minimize this probability to near zero.
